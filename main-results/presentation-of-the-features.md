The Cluster team selected ten authoring-tool features as the most relevant, useful, and feasible to ship for accessible-by-default publishing. Technical specifications describe the intended functionality for implementers. Source for this summary: 2024-03-03 archive page.

- Change language — quick access to set language changes within content.
- Tables creator — guided creation of accessible tables.
- Text alternative (ALT-text) — support, prompts, and AI suggestions for image descriptions.
- Forms editor — accessible, consistent form building with built-in safeguards.
- Video — requires accessible media and title support on embeds.
- Documentation — short contextual help with links to deeper guidance.
- Live testing while authoring — inline accessibility checks and remediation tips in the editor.
- Testing of documents — checks documents before publishing for automatic and manual issues.
- Testing the content of pages — block-level checks before publish to catch issues early.
- Testing the whole website — integrated testing with role-based filtering of results.
